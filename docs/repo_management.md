# Repo Management — Root ↔ Template Sync Policy

This document expands `repo_management_initial.md` to cover every file and directory in the
repository. It specifies, for each path, whether it must be **Copied**, **Aligned**, or
**Ignored** between the meta-repo root and the `template/` tree.

## Taxonomy

- **Copy** — The root file and its `template/` counterpart must be byte-identical. When the
  root file changes, mirror it into `template/` with `cp -r`. The `shared_hook_parity` check
  in `assets/root-template-sync-map.yaml` enforces this for shared hooks; use the same rule
  manually for every file listed as Copy below.
- **Align** — The files cover the same concern but legitimately differ (Jinja variables,
  project-specific dev deps, extra recipes). They must stay conceptually in sync: when you
  add/remove/rename a key in one, make the equivalent edit in the other. Use
  `just sync-check` (`scripts/check_root_template_sync.py`) and the targeted file-pair
  diffs to verify alignment.
- **Ignore** — The file is intentionally independent. Do not copy or reconcile it. Changes
  on one side should not trigger changes on the other.

## Global exclusions (never touch, never diff)

The following paths are always ignored regardless of which side they live on:

- `.venv/`, `.mypy_cache/`, `.pytest_cache/`, `.ruff_cache/`, `__pycache__/`
- `.git/`, `.coverage`, `.DS_Store`
- `temp*`, `temp_*/` (any ad-hoc scratch directory)
- `assets/` and `template/assets/` — user-customised after generation
- `docs/` and `template/docs/` — user-customised after generation
- `uv.lock` — regenerated by `just update`; never hand-edit or cross-copy
- `1` (stray scratch file at repo root) — delete if encountered

- `template/src/{{ package_name }}/**` — generated package source; no root counterpart.

## How to verify alignment

1. Run `just sync-check` — executes `scripts/check_root_template_sync.py` against
   `assets/root-template-sync-map.yaml`.
2. For Align pairs, diff the two files and confirm differences are limited to Jinja
   placeholders and project-specific content.
3. For Copy pairs, `diff -q` must return no output.
4. When adding a new file to either side, update this document and
   `assets/root-template-sync-map.yaml` together.
