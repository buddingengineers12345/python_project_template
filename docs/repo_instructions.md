# Repo Instructions — Step-by-Step Sync Guide

Purpose: A deterministic, task-by-task runbook for keeping the meta-repo root and the
`template/` tree in sync. Each TASK covers one folder or one logical group of files. Work
the tasks **in order**. Do not skip. Do not merge tasks.

## How to use this guide

Every TASK below follows the same fixed structure. A small model (e.g. Haiku) must:

1. Read **Scope** to confirm the task applies.
2. Read **Action legend** (once, below) and apply it.
3. Process each row of the **File table** left-to-right.
4. Run the commands in **Steps**.
5. Run the **Verification** block.
6. Only then advance to the next TASK.

### Action legend (applies to every TASK)

| Action | What to do | Command template |
|---|---|---|
| Copy | Overwrite the template file with the root file byte-for-byte. | `cp -f <ROOT> <TEMPLATE>` |
| Align | Compare both files. Differences must be limited to Jinja placeholders or project-specific values. If a key/recipe/section exists on one side but not the other, add the equivalent on the missing side. | `diff -u <ROOT> <TEMPLATE>` then manual edit |
| Ignore | Do nothing. Do not copy. Do not reconcile. Do not diff. | (none) |

### Global exclusions (never process in any TASK)

Always skip these paths, regardless of where they appear:

- `.venv/`
- `.git/`
- `.mypy_cache/`, `.pytest_cache/`, `.ruff_cache/`, `__pycache__/`
- `.coverage`
- `.DS_Store`
- `temp*` files and `temp_*/` directories
- `uv.lock` (regenerated by `just update`; never cross-copy)
- `assets/` and `template/assets/` (user-customised)
- `docs/` and `template/docs/` (user-customised)

## TASK 1 — Align root-level toolchain configs

**Scope:** Three files whose template counterparts use Jinja templating and therefore
cannot be byte-identical.

**Action:** Align (manual reconciliation).

**File table:**

| Path | Action | Template counterpart |
|---|---|---|
| `pyproject.toml` | Align | `template/pyproject.toml.jinja` |
| `justfile` | Align | `template/justfile.jinja` |
| `cliff.toml` | Align | `template/{% if include_git_cliff %}cliff.toml{% endif %}.jinja` |

**Steps:**

1. Run `diff -u pyproject.toml template/pyproject.toml.jinja` and reconcile new/renamed
   keys. Do not remove Jinja blocks (`{% if %}`, `{{ var }}`).
2. Run `diff -u justfile template/justfile.jinja` and reconcile recipe names, flags, and
   ordering.
3. Run `diff -u cliff.toml "template/{% if include_git_cliff %}cliff.toml{% endif %}.jinja"`
   and reconcile commit-type lists and section headers.

**Verification:** `just sync-check` completes without reporting pyproject / justfile /
cliff drift.

## TASK 3 — Copy env.example into Jinja envelope

**Scope:** `env.example` has a Jinja-wrapped counterpart; payload must match.

**Action:** Copy payload, preserve Jinja wrapper if present.

**File table:**

| Path | Action | Template counterpart |
|---|---|---|
| `env.example` | Copy (body) | `template/env.example.jinja` |

**Steps:**

1. Open `template/env.example.jinja`. If it has a Jinja header/footer, keep them.
2. Replace the body between the Jinja markers with the full contents of `env.example`.
3. If no Jinja markers exist, run `cp -f env.example template/env.example.jinja`.

**Verification:**

```bash
diff -u env.example template/env.example.jinja
```

Only Jinja-specific lines may differ.
